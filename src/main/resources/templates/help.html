Help Page

DATABASE: If you want to see the data stored in H2 database, use this link. (No password needed)

http://localhost:8080/console/

This project is using basic authentication, with in memory users. For testing purposes we are using User: marin Password: m123

Or as a header: Key: Authorization Value: Basic bWFyaW46bTEyMw==

For testing, follow these steps and use Postman, or some other similar tool.

1) Account

This is a POST request. Therefore, you will ned to select POST option in your Postman.

Headers: Key: Content-Type Value: application/json

URL: http://localhost:8080/register

Body: you will need to use JSON with key: "accountId", and value: e.g. "marin"

e.g.: t{ "accountId" : "marin" }

2) Register

This is a POST request.

Headers: Key: Content-Type Value: application/json, Key: Authorization Value: Basic bWFyaW46bTEyMw==

URL: http://localhost:8080/register

Body: you will need to use JSON with keys: "accountId", "redirectType" and value: e.g. "https://www.google.com/", 301

e.g.: { "url": "https://www.google.com/", "redirectType": 301 }

Response: { "url": "http://short.com/IhyiJo" } copy the value from response and use it in next step

3) Redirect

This is a GET request.

No need to use Postman for this one. Just call URL in your browser.

URL: http://localhost:8080/redirect?shortUrl={SHORT_URL}

e.g.: http://localhost:8080/redirect?shortUrl=http://short.com/IhyiJo

4) Statistic

Since we are using in memory users, this option will only return values if we provide "marin" as {accountId}

This is a POST request.

Headers: Key: Content-Type Value: application/json, Key: Authorization Value: Basic bWFyaW46bTEyMw==

URL: http://localhost:8080/statistic/{accountId}

e.g.: http://localhost:8080/statistic/marin

Response: JSON with values that are store in database.
